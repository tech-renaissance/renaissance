## 2. 代码层面的BUG分析

### 2.1 Philox RNG实现错误

**位置**: `include/renaissance/core/philox.h:150-156`
```cpp
TR_HOST_DEVICE TR_FORCEINLINE
float philox_uniform_float(uint64_t seed, uint64_t offset) {
    uint32_t r[4];
    philox_generate_4x32(seed, offset, r);

    // 标准方法：取高23位作为尾数
    constexpr float scale = 1.0f / 16777216.0f;  // 2^-24
    return static_cast<float>(r[0] >> 8) * scale;
}
```

**问题分析**：
- `r[0] >> 8` 取高24位，但scale是2^-24
- 这会导致生成的值范围严重偏离[0,1)

```cpp
}
```

## 6. 技术细节

### 6.1 当前Philox实现的具体错误

```cpp
// 错误的实现
constexpr float scale = 1.0f / 16777216.0f;  // 2^-24

// 问题：
// 1. r[0]是32位随机数
```

```cpp
// 正确的实现
constexpr float scale = 1.0f / 4294967296.0f;  // 2^-32
return static_cast<float>(r[0]) * scale;

```
