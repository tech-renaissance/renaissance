## 为什么IDCT缩放对ImageNet无效？

### 1. ImageNet图片尺寸偏小

**ImageNet数据集特点**:
- 大部分图片尺寸: 400×400 ~ 800×600
- 也有一些小图: < 500×500
- 很少有大图: > 1500×1500

**IDCT缩放的优势**:
- 对**大图**有效（如 1822×1024 → 224×224，1.44×加速）
- 对**中小图**无效（如 500×500 → 224×224，反而慢）

### 2. 二次SimdResize的开销

**问题**:
- IDCT缩放只支持固定比例: 1/2, 1/4, 1/8
- 大多数情况下，缩放后仍需二次SimdResize

**示例**:
```
原图 500×500 → 目标 224×224

Method 1 (直接SimdResize):
  500×500 → 224×224 (一次完成)

Method 2 (IDCT缩放 + SimdResize):
  500×500 → 250×250 (IDCT 1/2缩放)
  250×250 → 224×224 (Simd二次缩放)

结果: Method 2 需要两次操作，反而更慢!
```

### 3. IDCT缩放策略不够智能

**当前g.cpp的实现**:
```cpp
// 选择缩放后尺寸 ≥ 目标尺寸的最小缩放因子
if (scaled_w >= target_width && scaled_h >= target_height) {
    // 选择这个缩放因子
}
```

**问题**:
- 对于 500×500 → 224×224:
  - 1/2缩放: 250×250 (符合≥224的条件，被选中)
  - 但 250×250 → 224×224 的二次缩放开销很大

**更好的策略**:
- 选择缩放后尺寸**最接近**目标尺寸的缩放因子
- 或者设置阈值，只有当缩放比例 > 2×时才使用IDCT

### 4. Memory Copy开销

**Method 2的额外开销**:
```cpp
// 需要额外的temp_buffer存储IDCT缩放结果
uint8_t* temp_buffer_ = ALIGNED_ALLOC(SIMD_ALIGNMENT, temp_size);
tj3Decompress8(..., temp_buffer_, ...);  // IDCT缩放到temp_buffer
SimdResizerRun(resizer, temp_buffer_, ..., output_buffer, ...);  // 再从temp复制到output
```

对于小图，这次额外memory copy的开销相对较大。

## 结论与建议

### ✅ 结论

1. **IDCT缩放不适合ImageNet数据集**
   - 平均性能下降 **9.7%**
   - 83.7%的图片性能变差
   - 不推荐用于深度学习训练

2. **IDCT缩放仅适用于大图**
   - 输入尺寸 > 1500×1500
   - 输出尺寸较小（如缩略图生成）
   - 不需要精确尺寸匹配

3. **当前g.cpp的优化策略有缺陷**
   - 缺少尺寸阈值判断
   - 对小图也尝试IDCT缩放
   - 应该智能选择是否使用IDCT

### 📋 建议

**对于renAIssance框架**:
1. **不采用**IDCT缩放作为默认优化方案
2. **继续使用**f.cpp的Partial Decode方案（2.29×加速）
3. 如果需要支持大图，可以添加**智能判断**:
   ```cpp
   if (src_width > 1500 && src_height > 1500) {
       // 使用IDCT缩放
   } else {
       // 直接使用Full Decode + SimdResize
   }
   ```

**对于其他应用场景**:
- 缩略图生成：可以考虑IDCT缩放
- 超高分辨率图片处理：IDCT缩放有优势
- 图片压缩服务：根据实际场景测试选择
