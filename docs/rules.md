# 技术觉醒深度学习框架开发规范

## （一）本项目的命名空间叫tr。

## （二）代码中的所有输出和异常信息都只能用英文，不能包含中文和emoji。注释必须用中文，而且不能有emoji。

## （三）注释遵循doxygen规范，以方便以后自动文档生成。

## （四）主头文件是include/renaissance.h，其他头文件放在include/renaissance目录下的具体分类目录下。源文件在src目录下的具体分类目录下。项目测试样例在tests目录下的具体分类目录下。

## （五）renaissance.h主头文件要包含其他头文件，所有测试样例都只能包含renaissance.h和标准库，不能包含其他任何头文件。头文件用#pragma once避免重复包含。

## （六）代码中的调试输出、错误报告要用Logger类和TRException类，并且要正确区分日志级别和异常类型。此外，在报错时，一定要写明出错的类和函数，方便定位。

## （七）你接下来马上要做的四件事是：第一，打开根目录下的version.txt，看第一行了解我们当前的版本号。第二，打开docs/diary目录，阅读目录下的最新的两篇日志，了解最新进展；第三，阅读docs/logger_exception.md，了解日志器类和异常类的用法；第四，阅读docs/alpha_build.md，了解本项目推荐的编译方法。马上去做！！！
