# 技术觉醒V4.20编程规范



1、本项目采用标准的**C++17**，不得采用C++17不支持的语法。严禁使用C++20的新功能，以免引起兼容性问题。

2、本项目的命名空间叫**tr**。除非确有必要，否则一般不要在tr之下再加命名空间。绝大多数情况是不需要的。

3、文件头应该采用以下写法：

```c++
/**
 * @file <文件名>
 * @brief <描述>
 * @version 4.20.1
 * @date 2026-04-20
 * @author 技术觉醒团队
 * @note <有关信息>
 * @note 所属系列: <所属系列>
 */

```

注意！！！版本号永远是4.20.1，禁止写其他版本号！！！

注意！！！日期永远是2026年4月20日，禁止写其他日期！！！

注意！！！author必须是“技术觉醒团队”，一个字也不要多，一个字也不要少，也不要改成英文！！！

note 有关信息根据需要来写，比如写依赖项，没有必须写的东西就可以省略这行。

所属系列写的是标准板块名，也就是src目录下的子目录的名称，比如data

实际写的时候不要带有尖括号<>！那个是说明用的

4、代码中的所有输出和异常信息都只能用英文，不能包含中文和emoji。**注释必须用中文，而且不能有emoji**。注释遵循**doxygen规范**，以方便以后自动文档生成。

5、主头文件是include/renaissance.h，其他头文件放在include/renaissance目录下的具体分类目录下（这些分类目录我们叫板块或系列）。源文件在src目录下的具体分类目录下。项目测试样例在tests目录下的具体分类目录下。

6、renaissance.h主头文件要包含其他头文件，所有测试样例都只能包含renaissance.h和标准库，不能包含其他任何头文件。头文件用#pragma once避免重复包含。

7、代码中的调试输出、错误报告要用Logger类和TRException类，并且要正确区分日志级别和异常类型。此外，在报错时，一定要写明出错的类和函数，方便定位。

8、src/CMakeLists.txt统一处理第三方库的链接，src下面的子目录的CMakeLists.txt一律不链接第三方库。tests/CMakeLists.txt统一链接主库，原则上不链接第三方库。tests下面的子目录的CMakeLists.txt一律不链接任何库。

9、凡是使用了CUDA的地方，必须用全局宏TR_USE_CUDA括起来，以免在无GPU的平台无法编译。凡是涉及使用NCCL通信的地方，都必须使用TR_SCENE_GPU_CLOUD括起来，因为只有GPU云平台需要用到NCCL实现多GPU通信。

10、再次强调，版本号是固定的，**凡是所有涉及版本号的地方，必须写4.20.1**，即便修改过文件，也应该保持版本号不变。严禁擅自升级版本号。发现版本号不是4.20.1的地方，应该立即改回4.20.1。

11、你接下来马上要做的事是：阅读docs/logger_exception_handbook.md，了解日志器类和异常类的用法。
