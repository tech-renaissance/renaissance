# TR MLP Batch=128 三模式对比测试结果

**测试时间**: 2026-05-29  
**测试配置**: MNIST, 3-layer MLP (784→512→256→10), batch=128, seed=42, TF32 enabled  
**训练样本**: 60000 (train last batch = 96)  
**验证样本**: 10000 (val last batch = 16)  
**Epochs**: 3

## 三、关键观察

### 3.1 CPU vs GPU 差异趋势

| Epoch | CPU Val Loss | GPU Val Loss | 绝对差异 (Δ) | 相对差异 |
|-------|--------------|--------------|--------------|----------|
| 1     | 0.1335900    | 0.1335900    | 0.0000000    | 0.00%    |
| 2     | 0.0945500    | 0.0945489    | 0.0000011    | 0.001%   |
| 3     | 0.0875276    | 0.0881569    | 0.0006293    | 0.72%    |

**结论**: E0 完全一致，E1 微差，E2 差异放大。差异呈**累积性**，疑似训练过程中权重更新路径存在微小 divergence。

### 3.2 AMP (FP16) 数值漂移

AMP 模式从 E1 起即表现出与 FP32 基准的明显偏差：
- E1 val loss: 0.133503 (vs CPU 0.133590, 偏差 -0.000087)
- E3 val loss: 0.088382 (vs CPU 0.087528, 偏差 +0.000854)

这是 FP16 低精度累加的预期行为，但在本小型 MLP 上仍不可忽视。

### 3.3 性能对比

|------|---------------|-----------------|
| CPU  | ~2.7s         | 1.0×            |
| GPU  | ~0.4s         | **6.8×**        |

## 四、待排查项

1. **CPU/GPU E3 val loss 差异 (0.0875 vs 0.0882)**:  
   train loss 逐 epoch 一致，说明权重更新主干路径正确。差异可能来自：
   - val last batch (16 samples) 的 GPU 路径数值精度
   - `ACCUM_METRICS` float 累加 vs CPU `double` 累加的 79 次累积误差
   - 或 CUDA Graph last-batch variant (v5) 与 CPU dynamic launch 的算子行为差异

2. **AMP 训练 loss 漂移 (E2: 0.1006 vs 0.1004)**:  
   需确认 Gradient Scaling / Loss Scaling 逻辑是否对小型网络过于激进。
