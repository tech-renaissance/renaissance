## 主要成果

### 1. 新增trunc_cast_into方法 ✅
在cast_into（RNE舍入模式）基础上，新增trunc_cast_into方法，提供FP32→BF16的截断模式转换。

**设计动机**：
- RNE舍入虽然精度高，但需要额外的舍入逻辑，影响性能
- 在推理场景中，对精度要求不高，可以使用直接截断换取更高速度
- 提供用户选择：高精度（RNE）vs 高性能（Truncation）

**API设计**：
```cpp
void Device::trunc_cast_into(const Tensor& tensor_a, Tensor& tensor_b,
                             StreamType stream = TR_DEFAULT_STREAM);
```

**行为特点**：
- ✅ 仅支持FP32→BF16转换（其他类型组合抛出TypeError）
- ✅ 直接截断：丢弃FP32的低16位尾数
- ✅ 同步接口：函数返回时转换完成
- ✅ 形状必须相同：`tensor_a.shape() == tensor_b.shape()`

---

### 2. CPU实现：多架构SIMD ✅
在`src/device/cpu_cast.cpp`中实现trunc_cast_into的CPU版本。

**实现策略**：
- **X86_64 (AVX2)**：使用`_mm256_castsi128_si256`提取高16位
- **ARM64 (NEON)**：使用`vshrn_n_u32`窄化指令
- **RISC-V (RVV)**：使用`__riscv_vnsrl_wx_u16m4`右移窄化

**核心思想**：
- 统一采用SIMD向量化的位操作
- 直接右移16位，无舍入逻辑
- 性能与BF16→FP32相当（~4 G elems/s）

---

### 3. CUDA实现：内联汇编截断 ✅
在`src/device/cuda_cast.cu`中实现CUDA kernel。

**Kernel设计**：
```cuda
__global__ void k_fp32_to_bf16_trunc(const float* __restrict__ src,
                                     uint16_t* __restrict__ dst,
                                     size_t n) {
    size_t idx = blockIdx.x * blockDim.x + threadIdx.x;
    size_t stride = blockDim.x * gridDim.x;

    // 逐元素处理（不使用向量化，避免内存对齐问题）
    for (size_t i = idx; i < n; i += stride) {
        uint32_t bits;
        // 直接截断：丢弃FP32的低16位
        asm("mov.b32 %0, %1;" : "=r"(bits) : "f"(src[i]));
        dst[i] = static_cast<uint16_t>(bits >> 16);
    }
}
```

**关键改进**（Debug调试过程）：
- ❌ **初始版本**：使用`uint4`向量化写入
  ```cuda
  reinterpret_cast<uint4*>(dst)[i] = out;  // 非法内存访问
  ```
  **问题**：`dst`是`uint16_t*`，`uint4`是128位，向量化写入会越界访问

- ✅ **修复版本**：改为逐元素处理，避免内存对齐问题
  ```cuda
  for (size_t i = idx; i < n; i += stride) {
      dst[i] = static_cast<uint16_t>(bits >> 16);  // ✅ 安全
  }
  ```

**性能测试结果**（RTX 4060 Laptop，268M元素）：
| 转换模式 | 时间 | 吞吐量 | 加速比 |
|---------|------|--------|--------|
| **截断模式** | **7.94 ms** | **33.80 G elems/s** | **5.2x** |
| RNE模式 | 41.34 ms | 6.49 G elems/s | 基准 |

**关键发现**：
- 截断模式比RNE模式快**5.2倍**！
- 性能与BF16→FP32相当（33.69 G elems/s）
- 速度提升原因：直接截断无需舍入逻辑

### 4. MUSA实现：原生类型支持 ✅
在`src/device/musa_cast.cu`中实现MUSA kernel。

**Kernel设计**：
```cuda
__global__ void k_fp32_to_bf16_trunc(const float* __restrict__ src,
                                      __mt_bfloat16* __restrict__ dst,
                                      size_t n) {
    size_t idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        // 直接截断：使用内联汇编提取FP32的高16位
        uint32_t bits;
        asm("mov.b32 %0, %1;" : "=r"(bits) : "f"(src[idx]));
        // 右移16位，直接截断
        uint16_t bf16_bits = static_cast<uint16_t>(bits >> 16);
        // 通过memcpy避免类型双关
        __mt_bfloat16 result;
        std::memcpy(&result, &bf16_bits, sizeof(uint16_t));
        dst[idx] = result;
    }
}
```

**关键改进**（编译错误修复）：
- ❌ **初始版本**：使用`reinterpret_cast`转换右值
  ```cpp
  dst[idx] = reinterpret_cast<__mt_bfloat16&>(static_cast<uint16_t>(bits >> 16));
  // ❌ 错误：reinterpret_cast from rvalue to reference type '__mt_bfloat16 &'
  ```

- ✅ **修复版本**：使用`std::memcpy`避免类型双关
  ```cpp
  __mt_bfloat16 result;
  std::memcpy(&result, &bf16_bits, sizeof(uint16_t));
  dst[idx] = result;  // ✅ 符合C++标准
  ```

**编译通过**：MUSA平台编译成功

## 核心教训

### CUDA调试
1. **向量化写入陷阱**：
   - `uint4*`不能随意写入`uint16_t*`数组
   - 会导致越界访问和非法内存访问错误
   - 解决：逐元素处理更安全

2. **Debug模式的重要性**：
   - Debug模式下内存检查更严格
   - 能够发现Release模式下隐藏的问题
   - 使用`docs/alpha_build.md`的Debug配置非常有效

### MUSA调试
1. **类型转换规范**：
   - 不能对右值使用`reinterpret_cast`转换为引用类型
   - 使用`std::memcpy`是符合标准的类型双关方式
   - MUSA编译器比CUDA更严格

2. **原生类型支持**：
   - MUSA使用`__mt_bfloat16`原生类型
   - CUDA使用`uint16_t`存储BF16
   - 需要适配不同的类型系统

### 性能优化
1. **截断 vs RNE**：
   - 截断模式在GPU上提升显著（5.2x）
   - CPU上SIMD已经很快，提升有限
   - 推理场景强烈推荐使用trunc_cast_into

2. **用户选择权**：
   - 提供两种模式让用户根据场景选择
   - 高精度（RNE）vs 高性能（Truncation）
   - API设计清晰，不会误用
