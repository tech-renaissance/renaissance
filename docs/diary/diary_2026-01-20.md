## Quest #001: PARTIAL模式样本读取不完整问题 ✅ 已修复

### 问题现象
- **测试命令**: `--dts --val --lv 0 --mode partial --workers 8 --preprocess 2`
- **预期结果**: 50,000个样本
- **实际结果**: 8,049个样本 (16.1%)

### 根本原因
Preprocessor使用线性遍历`group_metas[0-3]`，消费完4个pair后退出，而IO线程已将所有26个logical pair循环加载到4个环形槽位。

### 修复方案
实现环形消费逻辑：`while(true)`循环 + `ring_idx = current_pair_idx % 4` + `logical_pair_idx`验证

### 测试结果
| 测试项 | 结果 |
|-------|------|
| **样本读取率** | 50,000/50,000 (100%) ✅ |
| **改进幅度** | +521% (从16.1%→100%) |
| **性能** | 5.45-5.92 GB/s |

### 性能问题修复
- **问题**: 使用`yield()`导致上下文切换爆炸，吞吐量<0.1 GB/s
- **修复**: 改用`sleep_for(100微秒)`，性能恢复到5-6 GB/s

## 跨Epoch随机可复现性 ✅ V3.8.2新增

### 实现内容

**1. 新增测试程序** (`test_reproducibility_epoch.cpp`)
- 测试目标：验证FULLY模式下多epoch的可复现性

**2. 改进现有测试** (`test_reproducibility.cpp`)
- 修改前：验证集shuffle硬编码为false
- 修改后：训练集和验证集都启用shuffle
- 测试目标：验证shuffle状态下的可复现性

**Shuffle状态可复现性** (PARTIAL模式，启用shuffle):
| 数据集 | 压缩级别 | 结果 |
|-------|---------|------|
| 验证集 | LV0 | ✅ 两次运行完全一致 |
| 验证集 | LV3 | ✅ 两次运行完全一致 |
| 训练集 | LV0 | ✅ 两次运行完全一致 |

### 设计要点

**可复现性保证公式**:
```
Level 2 (Block级):
  global_seed + epoch_id + is_train → seed → epoch_block_order[]

Level 3 (Sample级):
  global_seed + epoch_id + logical_pair_idx → shuffle_seed → shuffled_locations[]
```

**关键设计决策**:
- ✅ 验证集也支持shuffle（用于可复现性测试）
- ✅ 第一个epoch参与shuffle（`skip_first=false`）
- ✅ 不在核心逻辑中硬编码训练/验证集的区别
